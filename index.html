原码反码补码

正数源码：0000 0101
正数反码：0000 0101
正数补码：0000 0101

负数源码：1000 0101
负数反码：1111 1010
负数补码：1111 1011

正数补码 + 负数补码：1 0000 0000

浮点数二进制表示

任意一个二进制浮点数v可表示为：$v=(-1)^S\times2^E\times M$

S为符号位，E为阶码，M为尾数

float：32位，1S，8E，23M
double：64位，1S，11E，52M

float用E减去127表示实际阶码，如：0000 0001表示阶码-126
double用E减去1023表示实际阶码，如：0000 0001表示-1022

M表示小数位，如：0101……表示尾数1.0101……

特殊情况1：(E全为0)
此时M的0101……表示尾数0.0101……
float此时E表示阶码-126；double的E表示阶码-1022

特殊情况2：(E全为1)
M只能全为0，表示正负无穷大；不为0位无效数字

JavaSE基础

Posted on 2022-05-24 Edited on 2022-06-13 In java

类、对象

  • 类不是对象，是用来创建对象的模型

变量

  • 变量有两种：primitive主数据类型和引用。
  • 除非加上f，否则所有带小数点的值都会被当作double处理(如32.5f)。
  • 名称必须以字母、下划线或$开头。
  • 不存在对象变量，只有引用到对象的变量——即对象引用变量，保存的是存取对象的方法，类似指针和遥控器。
  • 声明一个Dog引用变量，此变量将永远被固定为Dog类型。
  • 数组也是对象。
  • 实例变量永远都会有默认值。(局部变量没有)
  • 实例变量是声明在类里，局部变量是声明在方法内。(局部变量必须初始化)
  • 变量比较：primitive主数据变量用==，对象用equals()

循环

for循环打标签

方法

  • 方法会运用形参。调用的一方会传入实参。(实参是传给方法的值。当它传入方案就成了形参)
  • java是通过值传递的，也就是说通过拷贝传递。(方法无法改变调用发所传入的参数)

from_utc_timestamp()

from_utc_timestamp(T a, STRING timezone)

Assumes(假设) given timestamp is UTC and converts(转换) to given timezone (as of Hive 0.8.0). For example, from_utc_timestamp(‘1970-01-01 08:00:00’,’PST’) returns 1970-01-01 00:00:00

to_utc_timestamp()

to_utc_timestamp(T a, STRING timezone)

Assumes given timestamp is in given timezone and converts to UTC (as of Hive 0.8.0). For example, to_utc_timestamp(‘1970-01-01 00:00:00’,’PST’) returns 1970-01-01 08:00:00.

扩展知识

GMT：Greenwich Mean Time 格林尼治标准时间。这是以英国格林尼治天文台观测结果得出的时间，这是英国格林尼治当地时间，这个地方的当地时间过去被当成世界标准的时间。

UT：Universal Time 世界时。根据原子钟计算出来的时间。

UTC：Coordinated Universal Time 协调世界时。因为地球自转越来越慢，每年都会比前一年多出零点几秒，每隔几年协调世界时组织都会给世界时+1秒，让基于原子钟的世界时和基于天文学（人类感知）的格林尼治标准时间相差不至于太大。并将得到的时间称为UTC，这是现在使用的世界标准时间。协调世界时不与任何地区位置相关，也不代表此刻某地的时间，所以在说明某地时间时要加上时区也就是说GMT并不等于UTC，而是等于UTC+0，只是格林尼治刚好在0时区上。GMT = UTC+0

时区简称(必须大写)

CST：西6区，UTC-6

PRC：东8区，UTC+8 (prc为中国简称。西6和东8简称都是cst，但函数会认为是西6)
